## 系統概述

五層架構個人 AI 知識庫。協助整理碎片資訊 → 結構化知識 → 實用輸出。
以 repo 實際資料夾結構為準。

---

## 五層資料夾架構

| 層級 | 資料夾 | 用途 |
|------|--------|------|
| 輸入 | `Notes/Clippings/` | Web Clipper 抓取 — **永不修改** |
| 輸入 | `Notes/Conversations/` | 有價值 AI 對話 — **永不修改** |
| 沉澱 | `知識庫/Concepts/` | 概念頁（按主題子目錄） |
| 沉澱 | `知識庫/Entities/` | 人物 / 組織 / 工具頁 |
| 沉澱 | `知識庫/Sources/` | 來源頁（按主題子目錄） |
| 沉澱 | `知識庫/index.md` | 知識庫總目錄 — 查詢第一站 |
| 沉澱 | `知識庫/log/` | AI 操作記錄（按月分檔） |
| 沉澱 | `LifeOS/` | 投資、健康、聯絡人等 |
| 輸出 | `Writing/` | 創作庫 / 影片筆記 / 教學素材 |
| 規則 | `Templates/` | 筆記固定格式 |

| 觸發詞 | Skill | 說明 |
|--------|-------|------|

## 嚴格規則

1. **先提案再執行** — 任何寫檔、移檔、補連結前，必須列方案等確認。
2. **先查 index 再建檔** — 新增頁面前必讀 `知識庫/index.md`，不建重複頁。
3. **不修改輸入層** — `Notes/` 原始內容永不改動。
4. **實際結構優先** — 與 repo 不符時以實際路徑為準。
5. **隔離邏輯** — 跨領域連結必須合理，不為連結而連結。
6. **Markdown 格式** — 內部連結用 `[[Page Name]]`。
7. **回答與執行分離** — 回答直接提供；寫檔相關必須先提案。
