## 背景

以范凱五層架構 + 戴倫自我進化 AI 知識庫為藍圖，對現有 Obsidian vault 進行全面改造。

## 改造前結構

```
Clippings/      ← 輸入（外部）
創作庫/          ← 輸入（自創）
影片筆記/        ← 定位不明
教學素材/        ← 定位不明
每日筆記/        ← 個人使用
知識庫/          ← 沉澱層
Templates/
CLAUDE.md       ← 規則（綁定 Claude）
```

**問題**：缺輸入統一入口、缺 AI 對話保存、缺輸出層整合、規則綁定 Claude、Ingest 全自動無確認機制。

## 決策紀錄

| 決策點 | 選擇 | 原因 |
|--------|------|------|
| 知識庫改名 Knowledge？ | 保留「知識庫」 | 避免破壞所有 Obsidian `[[]]` 連結 |
| 需要 LifeOS？ | 是 | 完整遵循范凱架構 |
| AI 確認機制？ | 是 | 范凱核心洞見：全自動會分類錯誤 |
| Conversations 目錄？ | 是 | AI 對話本身是知識資產 |
| 創作庫定位？ | 輸出層（Writing/） | 教案、教材屬於產出而非原始輸入 |

## 改造後結構

```
Notes/                 ← 輸入層
  Clippings/
  Conversations/
知識庫/                ← 沉澱層
  Concepts/
  Entities/
  Sources/
  index.md / log.md
LifeOS/                ← 沉澱層（生活）
  每日筆記/
Writing/               ← 輸出層
  創作庫/
  影片筆記/
  教學素材/
Templates/
AGENTS.md              ← 規則（工具無關）
```

## 具體變更

- 資料夾搬移 ×5（Clippings、每日筆記、創作庫、影片筆記、教學素材）
- 新建 Notes/Conversations/、LifeOS/、Writing/
- CLAUDE.md → AGENTS.md（加入 AI 確認機制、合併規則、Conversations 入庫規則、Writing 回流機制）
- 所有 Sources 的 `original_path` 更新為 `Notes/Clippings/...`
- index.md 來源欄路徑更新

## 關鍵洞見

- **范凱 vs Karpathy 最大差異**：AI 先列方案 → 人工確認（~10秒）→ 再執行。全自動會出現分類錯誤、重複建檔
- **輸出層存在的意義**：Karpathy 是研究者，沉澱就夠；教學內容創作者需要輸出層，且輸出完成後要回流到沉澱層形成閉環
- **AGENTS.md 而非 CLAUDE.md**：規則不應綁定特定 AI 工具，任何 AI 都能讀懂
