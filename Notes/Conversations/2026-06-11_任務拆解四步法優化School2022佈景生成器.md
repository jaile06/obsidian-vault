## 背景

以知識庫 [[任務拆解]]（Gary Chen 的 Human SOP → Agentic Workflow 四步法）檢視 `d:\workspace\theme_demo` 的 School2022 佈景自動生成器，對其 `/generate-theme` workflow（6 步）與 `xoops_theme_generator` SKILL.md 提出步驟優化方案。

---

## 四步法逐項體檢

### ① 格式標準化 — 基礎良好，文件漂移

已有 P0 MUST / P1 DEFAULTS（RFC 2119 式）與結構化 Markdown。缺口：

- **三份文件規則重複且不一致**：README 的輸出結構缺 `color_palette.json` / `theme_colors.json`（SKILL.md 有）；無前綴 logo 判斷 workflow 寫「寬度 < 800」、SKILL.md 只寫「寬度較小」
- **完全沒有 Error Handling 區段**：Step 4 exit code ≠ 0 時無任何分支定義
- 參數散落寫死：port 8765、縮圖尺寸、WCAG 4.5:1

### ② 任務拆解與連結 — 內部已拆，但 agent 與 script 有兩套重複邏輯

pipeline 已拆 6 支腳本、有 `color_palette.json` / `theme_colors.json` 兩個 artifact 串接。缺口：

- workflow Step 2 由 agent 用 `list_dir` + 比例規則自行分類圖片，與 `generate_theme.py` Step 0 的程式邏輯**重複** → 兩套判斷會漂移。應加 `--scan-only` 模式輸出 `scan_report.json`，agent 只讀 artifact（單一事實來源）
- WCAG 修了什麼是黑盒 → 應輸出 `contrast_report.json`（原值/新值/比值）
- Step 5 驗收只 `list_dir` 看目錄 → 應改讀 `manifest.json` 驗證各階段 artifact

### ③ 雙向開發 — 最大缺口：沒有迭代迴圈

- 使用者預覽後不滿意配色，唯一路徑是整包重跑（主色會被重新擷取覆蓋），又禁止手改 `output/` → 死路。應支援 `theme_overrides.json`（重跑不覆蓋）+ `--from-colors` 只重生 config/preview
- 默會知識回填：workflow 結尾應加「本次若有人工介入或失敗，把原因回填 SKILL.md Decision Rules」

### ④ 整合與執行環境 — checkpoint 不足

- 已有：Step 1 輸入確認、Step 6 預覽詢問
- 缺：(a) Step 3 AI 生圖後應讓使用者**確認圖片可用**再跑 pipeline（學校/政府網站對圖片敏感）；(b) `output/{theme_name}/` 已存在時確認覆蓋；(c) 無環境預檢（py / Pillow），不能 fail fast

---

## 優化後步驟（6 步 → 7 步）

| 步驟 | 內容 | 性質 |
|---|---|---|
| Step 0 | 環境預檢（py、Pillow）＋ output 覆蓋確認 | 新增 |
| Step 1 | 確認輸入，加 `theme_name` 格式驗證 `^[a-z0-9_]+$` | 強化 |
| Step 2 | 改呼叫 `generate_theme.py --scan-only` 讀 `scan_report.json` | 重構 |
| Step 3 | 補 AI 圖 ＋ 【checkpoint】使用者確認圖片 | 加 checkpoint |
| Step 4 | 執行 pipeline，定義成功判準與 Error Handling 分支 | 強化 |
| Step 5 | 讀 `manifest.json` + `contrast_report.json` 驗證回報 | 重構 |
| Step 6 | 預覽 ＋ 迭代迴圈（`theme_overrides.json` → `--from-colors`） | 加迭代 |

配套腳本修改：`--scan-only`、`contrast_report.json`、`manifest.json`、`theme_overrides.json` / `--from-colors`、統一三份文件規則（SKILL.md 為單一事實來源）。

P0 規則需同步修訂：「只能呼叫一次 generate_theme.py」→「`--scan-only` 與正式執行各一次，子腳本仍為 0 次」。
