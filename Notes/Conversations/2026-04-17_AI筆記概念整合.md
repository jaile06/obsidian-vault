## 整合分析

### 核心架構：第二大腦 × Raw-Wiki 雙層

[[第二大腦]] 是總體目標——用外部系統延伸記憶，越用越豐富。
[[Karpathy筆記方法]] 給出實作骨架：

```
Raw（原始不動）→ Wiki（AI整理）→ Index（查詢入口）
```

不需向量資料庫，靠 LLM 讀 Index 就能找到資料，適合個人 ~100 篇規模。

### 升級：五層架構（創作者版）

[[五層知識庫架構]] 在 Karpathy 上加了一層輸出層：

| Karpathy（研究者） | 范凱（創作者） |
|---|---|
| 輸入 → 沉澱 | 輸入 → 沉澱 → **輸出** |
| 知識存下來就夠 | 知識要變成教案、腳本、影片 |

關鍵差異：規則文件用 `AGENTS.md` 而非 `CLAUDE.md`，不綁定特定 AI 工具。

### 品質保障：確認機制 + 健康檢查

純自動化會出錯（分類錯誤、重複建檔），因此有兩道防線：

- **[[AI確認機制]]**（即時）：AI 先列方案 → 人看 10 秒確認 → 再執行
- **[[知識庫健康檢查]]**（定期）：每週掃孤立頁、斷裂連結、過薄頁面，輸出健康報告

### 閉環：自動入庫讓知識庫活起來

[[自動入庫]] 是關鍵的正向循環設計：

```
創作完成 → Claude 提取洞見 → 存回知識庫 → 下次創作有更豐富的庫可查
```

沒有這個回流，知識庫只是靜態存檔，不會成長。

### 整體框架

```
外部資料 ──→ Notes/Clippings ──→  知識庫  ──→ Writing（輸出）
AI對話   ──→ Notes/Conversations ↗  (沉澱)  ←── 自動回流
                                    ↓
                              每週健康檢查（自我修正）
```

## 關鍵洞見

- 五個概念不是並列的，而是有層次：**架構（Raw-Wiki）→ 擴充（五層）→ 守門（確認機制）→ 成長（自動入庫）→ 自癒（健康檢查）**
- 自動入庫是最容易被忽略但最關鍵的環節：沒有輸出回流，知識庫是單向消耗，不會形成複利
