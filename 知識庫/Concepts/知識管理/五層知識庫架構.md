## 定義

范凱在 Karpathy 三層架構（Raw/Wiki/Rules）基礎上擴充的五層知識庫，新增輸出層，適合內容創作者與軟體開發者。

## 核心重點

| 層級 | 資料夾 | 用途 |
|------|--------|------|
| 輸入 | Notes | 所有輸入的唯一入口，不用分類直接丟 |
| 沉澱 | Knowledge | 自己寫的原創知識與方法論 |
| 沉澱 | LifeOS | 投資、健康、聯絡人等生活工作資料 |
| 輸出 | Software | 工具技巧、軟體產品文件 |
| 輸出 | Writing | 視頻腳本、文章、運營 SOP |

- **規則**：統一寫在 AGENTS.md，不綁定特定 AI 工具
- **Conversation 目錄**：與 AI 的有價值對話本身也是知識資產

## Vault 改造決策紀錄（2026-04-17）

| 決策點 | 選擇 | 原因 |
|--------|------|------|
| 知識庫改名 Knowledge？ | 保留「知識庫」 | 避免破壞所有 `[[]]` 連結 |
| 需要 LifeOS？ | 是 | 完整遵循范凱架構 |
| AI 確認機制？ | 是 | 全自動會分類錯誤（范凱核心洞見） |
| Conversations 目錄？ | 是 | AI 對話本身是知識資產 |
| 創作庫定位？ | 輸出層（Writing/） | 教案、教材屬於產出而非原始輸入 |
| 規則檔命名？ | AGENTS.md（非 CLAUDE.md） | 規則不應綁定特定 AI 工具 |

## 不同視角 / 矛盾之處

- 比 Karpathy 原版多了「輸出層」；Karpathy 是研究者，沉澱就夠；范凱是創作者，知識要變成產出
