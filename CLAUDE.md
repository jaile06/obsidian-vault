# 我的第二大腦 — CLAUDE.md

## 關於我
- 我是程式科資訊組長
- 這個 vault 是我的教學第二大腦

## 語言偏好
- 所有回應請使用繁體中文
- 筆記內容也用繁體中文

## 筆記庫結構（三層）

| 資料夾 | 用途 | 存什麼 |
|---|---|---|
| `Clippings/` | 輸入 | 網路上剪藏的文章、影片筆記、別人的資料 |
| `知識庫/` | 消化 | AI 整理後的結構化知識（有 index.md 索引） |
| `創作庫/` | 輸出 | 我自己的教案、教材、報告、原創內容 |
| `每日筆記/` | 時間管理 | 每日紀錄、週計畫、知識重整週報 |
| `Templates/` | 模板 | 各種筆記的固定格式 |

## 工作規則

### 回答風格
- 回答要簡潔，不要冗長
- 每次回答都要附上建議（下一步行動或改善方向）

### 新增筆記時
- 一律加上 frontmatter（title、date、tags）
- 根據內容判斷存到哪個資料夾：
  - 別人的文章 → `Clippings/`
  - AI 整理的知識 → `知識庫/`
  - 我自己的作品 → `創作庫/`
  - 每日/每週紀錄 → `每日筆記/`

### Clippings 處理規則
- 使用者透過 Web Clipper 剪藏的文章會自動存到 `Clippings/`
- 這些是原始資料，**不要修改**
- 每週知識重整時，AI 會把 Clippings 消化成知識庫頁面

### 知識庫維護規則
- 知識庫有 `index.md`（目錄）和 `log.md`（操作紀錄）
- 每次新增知識庫頁面，都要更新 index.md
- 每次做知識重整，都要在 log.md 新增紀錄

### 使用者說「幫我新增到筆記」時
- 代表存到這個 Obsidian vault
- 根據內容判斷存到正確的資料夾
